Declaration Syntax

Grammatically, a declaration consists of two components, a set of declaration specifiers and a comma-separated list of declarators, terminated with a semicolon. The declaration specifiers indicate a particular qualified base type, storage duration, and linkage characteristics that the subsequent declarators are to be derived from. Following the declaration specifiers, the declarator list is a list of identifiers to be declared. Each declarator may include additional type derivation and object initialization for that specific identifier.

Declaration Specifiers

A declaration must specify a valid qualified base type and may also be preceded by an optional function and/or storage class specifier. While there are no restrictions on the order that these components are written, students should practice the following format,

[storage-class-specifier] [function-specifier] type-specifier... [type-qualifier...]

Base Type Specification

The qualified base type is denoted by a combination of type specifiers, which name a type, and type qualifiers, which make or break certain promises made to the implementation.

Type Specifiers

The type specifiers designate the base type from which each declarator will be derived; this may be the name of a scalar type, the name of a type alias, a structure, a union, or an enumeration.

Absent from this list are the pointer, array, and function types, which cannot be derived as part of the type specifiers, but are instead separately derived in each declarator, as described below.

Type Qualifier

The type denoted by the type specifier may be const, volatile, and/or restrict qualified–called cvr-qualification.

Additional Specifiers

The linkage and storage durations of identifiers and objects declared can be altered by the storage class and function specifiers. Additionally, the typedef storage class specifier transforms function or object declarators into type alias declarators.

Storage Class Specifier

There are five storage class specifiers, auto, extern, register, static, and typedef.

Function Specifier

There is only one function specifier, inline, that is used for function inlining.

Declarators

Following the declaration specifiers, a comma-separated list of declarators introduce identifiers for objects and functions, or corresponding type aliases when the typedef specifier is used. Each declarator may individually derive pointer, array, and function types from the base type in the declaration specifiers. The declarator grammar allows for recursive, repeated derivation of the pointer, array, and function types from already derived types,

Grouping Construct

A declarator may be enclosed in parentheses,

(declarator)

Pointer Derivation

A declarator may be modified by prepending an asterisk (*) and optional type qualifiers, to derive a pointer type,

[* [type-qualifier...]] declarator

For example, char *s;.

Array Derivation

A declarator may be modified by appending square brackets, optionally enclosing a size expression, to derive an array type,

declarator[[size-expr]]

For example, int letter_freq[26];

If a size expression is not provided, the array type is incomplete.

Function Derivation

A declarator may be modified by appending parentheses, optionally enclosing a list of types and/or declarations representing parameters, to derive a function type,

declarator([type-or-declaration-list])

For example, int sum(int a, int b).

Initializers

A declarator for an object may also provide an initial value as an assignment expression for non-aggregate types, in the form,

= expression

and for aggregate types (arrays and structures), in the form,

= { initializer-list }

When initializing arrays and structures, each initializer in the initializer list provides the value of each successive element or member. Specific elements or members may be designated with either

[ index ] =

for an array, or

. member-identifier =

for a structure.
