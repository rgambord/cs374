Exit Status

The exit status of your program is the integer value passed to the exit() standard library function. Returning from main() with a value also implicitly calls exit()–under the hood, the C runtime implements this by directly passing the return value of main() to the exit() system call, as in exit(main()). Additionally, if the end of the main function is reached without an explicit return statement, or a call to exit(), it is equivalent to exiting with a value of zero.

The standard library also exposes the macros EXIT_SUCCESS and EXIT_FAILURE as portable exit statuses. POSIX further defines the exit status of a program as the “lower eight bits” of the integer exit status passed to exit()–in other words, a value in the range 0 - 255. The value 0 indicates successful program execution and termination, while a non-zero value indicates an error. Thus EXIT_SUCCESS == 0, and EXIT_FAILURE != 0 (typically 1).

Each of the following programs is equivalent,

#include <stdlib.h>

int
main()
{
  if (1 + 1 == 2) {
    exit(EXIT_SUCCESS);
  } else {
    exit(EXIT_FAILURE);
  }
} /* Implicit return EXIT_SUCCESS -> exit(EXIT_SUCCESS) */

#include <stdlib.h>

int
main()
{
  if (1 + 1 == 2) {
    return EXIT_SUCCESS; /* Implicit exit(EXIT_SUCCESS) */
  } else {
    return EXIT_FAILURE; /* Implicit exit(EXIT_FAILURE) */
  }
} /* Implicit return EXIT_SUCCESS -> exit(EXIT_SUCCESS) */

#include <stdlib.h>

int
main()
{
  if (1 + 1 == 2) {
    /* Fallthrough to end */
  } else {
    return EXIT_FAILURE; /* Implicit exit(EXIT_FAILURE) */
  }
} /* Implicit exit(EXIT_SUCCESS); */
